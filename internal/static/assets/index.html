账号导入

支持 JSON 对象、JSON 数组和 NDJSON。文件内容会先回填到下方文本框，导入前会做基础校验，并使用现有 /stats 数据做重复邮箱预检查。

上传文件

选择本地账号文件后，不会立即上传，而是先读取为文本，便于你在提交前确认与编辑。

拖拽暂不支持，点击按钮选择本地文件 推荐使用 JSON 对象、JSON 数组或 NDJSON。提交时会发送标准 JSON 文本请求体，因此这里不会使用 multipart/form-data。

导入内容

你可以直接粘贴账号数据，也可以在读取文件后继续手工修改再提交。

校验与提示

提交前会检查格式、最小凭证字段、缺少 email 的记录，以及是否与当前账号池中的邮箱重复。

最近一次导入结果

导入完成后，统计数据会自动重新拉取。你也可以返回统计页查看最新列表。

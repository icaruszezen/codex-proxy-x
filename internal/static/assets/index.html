账号导入

支持 JSON 对象、JSON 数组、NDJSON，以及 sub2api 多账号 JSON 数组。文件内容会先回填到下方文本框，导入前会做基础校验，并使用现有 /stats 数据做重复邮箱预检查。

上传文件

选择本地账号文件后，不会立即上传，而是先读取为文本，便于你在提交前确认与编辑。

拖拽暂不支持，点击按钮选择本地文件 推荐使用 JSON 对象、JSON 数组、NDJSON 或 sub2api 导出的多账号 JSON。提交时会发送标准 JSON 文本请求体，因此这里不会使用 multipart/form-data。

导入内容

你可以直接粘贴账号数据，也可以在读取文件后继续手工修改再提交。

校验与提示

提交前会检查格式、最小凭证字段、缺少 email 的记录，以及是否与当前账号池中的邮箱重复。

错误明细

    账号登录（OAuth 粘贴回调）

    远程部署无法回调本地 1455 端口时，可使用此流程：先在后端生成授权链接，登录后把浏览器地址栏中的回调 URL 粘回，由后端用 PKCE 直接换取 Token 并加入号池。

    第 1 步：生成授权链接

    点击下方按钮，由后端生成带 PKCE 的 OpenAI 登录链接，链接 10 分钟内有效。

    第 2 步：粘贴回调地址

    登录完成后，浏览器会跳转到形如 http://localhost:1455/auth/callback?code=...&state=... 的地址（即使页面打不开也没关系，把整段地址从地址栏复制下来粘贴到这里即可）。

    qmsg 私聊通知

    配置 qmsg 私聊推送通道。启用后，账号发生自动删除或自动停用事件时，会通过 qmsg 发送通知。

    未加载

    KEY 保存后会写入服务端本地配置文件；重新打开页面时只显示掩码。

    当测试返回“您选择的Qmsg不在线”时，可清空此项或换一个在线机器人 QQ。

    可用变量：{{action}}、{{email}}、{{reason_code}}、{{detail}}、{{storage_mode}}、{{timestamp}}、{{timestamp_utc}}。
